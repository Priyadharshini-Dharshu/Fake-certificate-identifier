Project Report: Certificate Authenticity Verification System

1. Project Title

Certificate Authenticity Verification System

2. Project Objective

The primary objective of this project is to develop an automated, intelligent web application capable of verifying the authenticity of digital or scanned certificates. By utilizing Computer Vision, Optical Character Recognition (OCR), and a rule-based AI scoring mechanism, the system aims to detect anomalies, text patterns, and image tampering, thereby preventing credential fraud efficiently.

3. Problem Statement

With the rapid digitization of academic and professional credentials, certificate forgery has become increasingly prevalent and sophisticated. Manual verification of certificates is a time-consuming, error-prone, and resource-intensive process, making it impractical for large organizations, universities, and employers. There is a critical need for an automated, reliable, cost-effective, and fast system to detect fake certificates and authenticate genuine ones without relying solely on centralized databases or manual background checks.

4. Technologies Used

  • Backend Framework: Python 3.10+, Flask
  • Frontend Technologies: HTML5, CSS3, Bootstrap (from templates)
  • Computer Vision & Image Processing: OpenCV, NumPy
  • Optical Character Recognition (OCR): Tesseract OCR, PyTesseract, Pillow (PIL)
  • Machine Learning & Analytics: Scikit-Learn (dependency), custom rule-based heuristics engine, and weighted feature models
  • Database/Storage: JSON (local file storage for authentication and verification logs)

5. System Architecture Explanation

The system follows a classic Client-Server Architecture.

  • Client Side: The frontend provides a responsive web interface where users can upload certificate images. It also features an admin dashboard for navigating historical verification reports.
  • Server Side: The Flask backend handles all core processing and routing. Upon receiving an image upload, it saves the file locally.
  • Processing Pipeline: The image is sent through an Image Preprocessing Module (OpenCV) for contrast enhancement and denoising. It then branches into two parallel paths:
    1. OCR Text Extraction: Tesseract extracts textual content to check for mandatory certificate keywords.
    2. Feature Extraction: OpenCV computes visual metrics (sharpness, aspect ratio, text density, edge density, entropy, uniformity, and contrast).
  • AI Prediction Engine: Both textual and visual features are evaluated by a weighted rule-based model. It calculates an authenticity percentage and makes a definitive classification (REAL or FAKE).
  • Storage: Verification results and user data are permanently stored in local JSON files (reports.json and users.json).

6. Detailed Module List

Web Interface Module (Frontend)

  • Purpose: Provides the user interfaces for certificate upload, viewing results, and admin dashboard operations.
  • Files used: templates/index.html, templates/admin.html, templates/login.html, templates/about.html, templates/services.html, templates/contact.html
  • Working explanation: Built with Bootstrap, these templates render dynamic GUI pages via Flask. They handle file upload via forms, send asynchronous requests, and visualize the verification summary (including confidence bars) directly in the user's browser without full page refreshes.

Authentication Module

  • Purpose: Secures administrative functions and the reporting dashboard.
  • Files used: app.py, users.json
  • Working explanation: Employs Flask sessions to maintain login states. When an administrator attempts to access routes like /admin or /api/reports, a @login_required decorator validates their session against credentials loaded from users.json.

Core OCR & Preprocessing Module

  • Purpose: Prepares uploaded images and extracts embedded text accurately.
  • Files used: app.py (extract_text_from_image function)
  • Working explanation: Utilizes OpenCV to convert images to grayscale, applies CLAHE (Contrast Limited Adaptive Histogram Equalization), uses FastNlMeansDenoising, and applies Otsu's thresholding. Subsequently, PyTesseract runs over the thresholded image to extract machine-readable text.

Feature Extraction Module

  • Purpose: Analyzes visual characteristics and identifies potential digital tampering.
  • Files used: app.py (analyze_certificate_features, detect_uniform_regions)
  • Working explanation: Evaluates the image's aspect ratio and sharpness (using Laplacian variance). It computes text density relative to image dimensions, measures information entropy, calculates edge density using the Canny edge detector, and detects uniform regions that strongly indicate copy-paste digital manipulation.

AI & Prediction Module

  • Purpose: Aggregates extracted metrics to determine if a certificate is authentic or fake.
  • Files used: app.py (predict_authenticity)
  • Working explanation: Analyzes the extracted text for over 20 mandatory keywords (e.g., "awarded", "signature", "seal"). It uses a sophisticated weighted scoring mechanism—adding points for high sharpness, adequate entropy, and passing keywords; and deducting points for low density, high uniform regions, and missing keywords. A final confidence score and decision are generated.

Reporting & Storage Module

  • Purpose: Logs verification history for administrative auditing.
  • Files used: app.py, reports.json
  • Working explanation: Serializes each verification attempt (including raw features, statistics, filename, and final verdict) into a JSON object and appends it to reports.json. It exposes API endpoints for the admin panel to retrieve, clear, or selectively delete these reports.

7. Step-by-Step System Workflow

  1. Upload: A user accesses the homepage and uploads a scanned certificate image.
  2. File Validation: The backend checks for valid extensions (PNG, JPG, PDF-converted, etc.) and file size limits.
  3. Local Save: The file is securely saved to the static/uploads/ directory with a timestamp identifier.
  4. Preprocessing: The image is loaded, converted to grayscale, enhanced with CLAHE, denoised, and thresholded.
  5. OCR Processing: Tesseract extracts structural text from the clean image.
  6. Feature Extraction: Visual metrics like image sharpness, aspect ratio, edge density, entropy, contrast, and uniform regions are calculated from the original image.
  7. Evaluation (AI Model): The system parses the extracted text for specific semantic certificate keywords. It aggregates the keyword count and the visual features through a logical weighted rule-based heuristic.
  8. Scoring: A final unified score is calculated. If the normalized "real percentage" is >= 40%, the certificate is classified as REAL; otherwise, it is FAKE.
  9. Logging: The prediction, calculated metrics, and file attributes are recorded locally in reports.json.
  10. Result Display: A JSON response is sent back to the frontend, which renders the verdict, a confidence bar, and specific reasons for the generated result to the user.

9. Algorithm & Method Used

The project employs a Heuristic Rule-Based Scoring Algorithm that combines both Optical Character Recognition (OCR) and Computer Vision Features.

  • Textual Semantic Analysis: Uses NLP paradigms to identify a list of over 20 specific, high-value keywords (e.g., "certificate", "awarded", "diploma", "signature").
  • Computer Vision Analysis:
    • Sharpness & Clarity: Computes the Variance of Laplacian. A low variance indicates a heavily blurred/unreadable input.
    • Tamper Detection: Identifies contiguous uniform areas (where the Standard Deviation of pixel intensity patch is < 5). High frequencies of these patches usually indicate edited, erased, or heavily compressed copy-pasted regions.
    • Information Entropy: Calculates Shannon Entropy via histogram probability distributions to measure the true informational complexity of the certificate structure.
  • Aggregated Decision Modeling: The system uses a threshold logic system. Starting with a normalized base score, it aggregates positive points for favorable feature values (such as >3 keywords, high sharpness, ample edge density) and subtracts heavy penalties for anomalies (such as low textual content, suspicious uniform regions, or an unnatural aspect ratio).

10. Verification Logic to Detect Fake Certificates

The exact logic flow ensuring authentication relies on strict conditional fallbacks:

  1. Critical Keyword Check: If the image completely lacks certificate-related terminology, it is instantly penalized heavily (-50 points) and classed as FAKE.
  2. Content Volume Verification: The system validates if there are sufficient independent text blocks (minimum 5). Fewer blocks result in a penalty for insufficient context.
  3. Visual Quality Penalty: It deducts points actively for poor image sharpness (< 50) and low image entropy (< 4), as faked certificates often rely on low-quality reprints to obscure digital flaws.
  4. Copy-Paste Anomaly Mapping: If the system detects suspicious uniform regions (> 20 occurrences of static standard deviation), it heavily penalizes the score by 15 points, signaling potential structural tampering.
  5. Final Metric Calculation: A normalized percentage is tabulated: ((score + 100) / 200) * 100. A passing threshold of 40% is enforced. Results above this are deemed REAL, and results below are structurally flagged as FAKE alongside detailed trace reasons appended individually (e.g., "Unusual aspect ratio", "Low edge density").

11. Advantages of the System

  • Rapid Automation: Reduces verification checking time from days to mere seconds per document.
  • Multi-modal Cross-Checking: Defends against fraud by checking both contextual meaning (text) and digital manipulation artifacts (vision attributes).
  • High Usability: Offers a visually appealing, zero-configuration bootstrap web interface featuring immediate visual feedback and confidence indicators.
  • Administrative Traceability: Dedicated dashboard reliably keeps historical records of all verifications for auditing purposes.
  • Cost-Effective: Operates entirely autonomously using local, robust open-source libraries (OpenCV, Tesseract) without relying on expensive paid external APIs.

12. Limitations

  • Rule-Based Constraints: Relies on rigid heuristics, which might inadvertently penalize beautifully designed but highly unconventional real certificates.
  • Resolution Sensitivity: Tesseract OCR accuracy degrades rapidly on very low-resolution or heavily compressed uploads, leading to potential false negatives.
  • Structural Bias: The logic strongly expects a traditional diploma structure, lacking the flexibility to evaluate heavily stylized modern digital badges.
  • Exclusively English: The OCR configuration and hard-coded keyword heuristics limit correct operations strictly to English language certificates.
  • No Cryptographic Backing: It analyzes surface-level sensory characteristics rather than verifying a digital blockchain hash or cryptographic signature at the issuer source.

13. Future Enhancements

  • Deep Learning Transition: Replace the static rule-based system with a Convolutional Neural Network (CNN) trained on vast datasets of both authentic and counterfeit documents.
  • Barcode & QR Integration: Add dedicated scanning modules to directly decode embedded visual links.
  • Multilingual OCR Support: Integrate localized datasets to verify international certificates accurately.
  • Issuer API Connectivity: Implement direct API verifications into university databases or blockchain ledgers to perform active cross-referencing.
  • Cloud Scalability: Containerize (Docker) and deploy to a cloud infrastructure (AWS/Azure) to effortlessly manage concurrent high-load processing instances.
